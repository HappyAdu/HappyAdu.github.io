jQuery性能优化

发表于 2016-11-03   |   分类于 前端   |     |   阅读次数

  对于前端初学者而言，学习jQuery是必须经过的一步，毕竟这门相对老牌的js库许多的项目现在仍在使用。一是这门语言容易上手，且几乎人人都会，可以减少团队成员的学习成本（人人都会），一是这个库的确也提供了大量可靠的插件扩展，能提高我们的开发效率。那么如何什么样的jQuery代码性能最佳？就我个人观点是，尽量减少DOM操作，尤其是那些可能会引起页面重绘的操作是最佳实践不可或缺的一项法则。其次，便也是减少全局对象的直接引用了。

那么，具体细节，客观随我一观：

1.使用性能较优的选择器
  和原生JS想比，jQuery取对象的方式简单易学，但是什么样的选择器更快呢？曾经的一段时间我一度以为是id,但后来看了些博客后我知道我是错了。在JQuery中 选择器性能排序是这样的： id  >  标签选择器  >  属性选择器 > \$(“:hidden”)等伪选择器。建议尽可能使用id选择器，标签选择器作为第二优先选择，有选择的使用类。尽可能避免使用后续方法选择。
2.将对象缓存

10.使用链式写法
jQuery的一大特点，就是允许使用链式写法。

采用链式写法时，jQuery自动缓存每一步的结果，因此比非链式写法要快。根据测试，链式写法比（不使用缓存的）非链式写法，大约快了25%。
11.减少DOM操作
（1）改动DOM结构开销很大，因此不要频繁使用.append()、.insertBefore()和.insetAfter()这样的方法。
如果要插入多个元素，就先把它们合并，然后再一次性插入。根据测试，合并插入比不合并插入，快了将近10倍。
（2）如果你要对一个DOM元素进行大量处理，应该先用.detach()方法，把这个元素从DOM中取出来，处理完毕以后，再重新插回文档。根据测试，使用.detach()方法比不使用时，快了60%。
